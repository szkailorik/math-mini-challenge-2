# 计算能力大师系统 v3 · 设计文档

日期：2026-07-10 ｜ 状态：已经用户逐节确认 ｜ 承载：math-mini-challenge-2 原仓库大版本升级

## 0. 背景与调研依据

v2 是"巅峰收束段"单阶段系统。用户要求回到计算的本质，做覆盖**全小学+初中衔接**的长期训练系统，解决四个痛点：题型覆盖不全、习题感不足、变式不贴合错因、错题本出卷不够快。

设计依据（调研报告要点，来源见调研记录）：

1. 课标（2022版）只对「20以内加减、表内乘除、百以内简单口算」要求"熟练"→ 自动化清单窄而明确；运算能力是唯一贯穿四学段的核心素养。
2. 流利度实证：自动化 = 2 秒内提取；达标线 = 正确率≥95% 且约 30–40 题/分钟；**每天 2–5 分钟计时提取**优于每周集中练；已达标仍需低频维持。
3. 认知科学（Buggy/Repair Theory）：55%+ 的计算错误是**系统性 bug**（可诊断、可预测），错题应按错因建档，而非按"粗心"。
4. 交错练习 RCT：交错出题 + 陷阱辨别题比分块刷题留存高 20–40%；简算训练必须混入"长得像能简算但不能"的题。
5. 简算家族：校内 8 类（凑整/加法结合/乘法结合/分配正逆/减法性质/除法性质/拆数补偿/带符号搬家）+ 拓展 3 类（基准数/等差求和/裂项）。
6. 分数小数混合四则是小学计算"珠峰"，错误须回溯到 3 个子技能：互化时机判断、统一表示策略、交叉约分。
7. 方程走**等式性质**路线（衔接初中移项），补 x 在减数/除数位、两侧含 x 题型。

## 1. 目标与非目标

**目标**：两个学员（KAI/Lorik）长期（至小学毕业）高质量训练计算能力；家长日操作 ≤5 分钟（打印+批阅录入）；错题两次点击出专项训练册。

**非目标**：应用题/几何/读题训练；奥数竞赛级巧算（裂项等仅作拓展层）；移动原生应用；多于 2 个学员的机构化。

## 2. 技能地图（js/map/skills.js）

约 60 个原子技能点，三层：

### 地基层（fluency，约 12 点）——每日计时，永不毕业
add20 / sub20（20以内进退位）、mult_table / div_table（表内）、add100 / sub100（百以内口算）、mult_pairs（凑整对 25×4、125×8、5×2）、dec_shift（小数×÷10/100/1000 移位）、dec_frac_base（基准互化 1/2、1/4族、1/5族、1/8族、1/20、1/25、百分数）、frac_same_denom（同分母加减口算）、square_base（常用平方数，拓展）、estimate_flash（数量级速估）。

达标线：正确率 ≥95% 且速度达标（双轨：30–40题/分钟基础事实线 + 中式年级区间线）。达标后进入**维持态**（出现频次降为 1/4），错一次立即回到在练态。

### 结构层（procedure，约 30 点）——集中练透 + 间隔回访
整数：多位数加减（含连续退位/中间有0）、乘一位数、乘两位数、除一位数、除两位数、余数与验算。
小数：意义与比较、加减、乘法（定位）、除法（同倍扩大/商<1/循环）、近似取舍。
分数：意义与性质、通分、约分（含交叉）、同/异分母加减、乘法、除法、带分数转换、假分数。
通用：运算顺序（含中括号）、估算策略、验算链。
管理：新技能点先小步分块（连续 2 次模块 ≥90% 视为练透）→ 转入间隔回访（间隔 4→8→16 套指数拉长，回访 <80% 则间隔减半并回到在练态）。

### 策略层（strategy，约 18 点）——交错训练 + 陷阱辨别
简算 8 校内家族各为一点 + 拓展 3 点（默认关闭，可在设置开启）；方程 6 型（x±a=b、ax=b/x÷a=b、ax±b=c、a(x±b)=c、a−x=b/a÷x=b、两侧含x）；混合四则 3 子技能（互化判断、统一表示、交叉约分）+ 综合混算 2 点（整小混合、分小混合珠峰）；单位率关系保留 v2 的 4 点。
策略层模块**强制交错**：同模块内不同家族混排，简算模块含 25% 陷阱对比题。

每个技能点字段：`{ id, layer, label, gradeBand, prereqs[], graduation:{acc, speed?}, revisit:{base, max} }`。依赖图必须无环（CI 校验）。

## 3. 题模库（js/qbank/*.js，首期约 120 个题模）

题模 = 声明式规格 + 生成函数，一个题模对应一个技能点的一种题型档位：

```js
{
  id: 'smart.dist_reverse.basic',
  skill: 'smart.dist_reverse',
  tier: 'basic' | 'core' | 'peak',      // 同技能点内的档位（替代 v2 的 level 2/3/4）
  traps: ['漏乘一项', '除法分配到加法'],  // 本题模设计考察的坑（文档性）
  bugs: ['dist.partial', 'dist.over_div'], // 关联 bug 库 id
  variantLadder: { L2: 'reparam', L3: 'structure', L4: 'contrast' },
  generate(rng) => Question,             // Frac 精算答案，含 hint 与 steps
  variant(rng, bugId|null, level) => Question, // 按 bug/阶梯出变式
}
```

Question 在 v2 基础上增加 `skill`、`qmodel`、`isTrap`（陷阱对比题标记）、`steps`（讲解用分步，可选）。

### bug 库（js/map/bugs.js，约 40 条）
`{ id, label, diagnose(一句话话术), explain(讲解要点), family: 知识性|技能性|策略性 }`。
示例：`sub.borrow_chain`（连续退位断链）、`dec.point_mult`（积的小数位数错）、`dec.point_div`（商定位错）、`frac.reduce_cross`（交叉约分漏约）、`order.same_level`（同级未从左到右）、`dist.partial`（分配漏乘）、`paren.sign`（去括号不变号）、`smart.fake`（把不能简算的当能简算）、`eq.move_sign`（移项不变号）等。

批阅界面：标 ✗/？ 后浮出该题模关联的 2-3 个 bug 按钮（一次点击，可跳过）。标了错因 → 错题条目带 bugId → 变式按 bug 配方出；未标 → 按题模默认阶梯。

## 4. 掌握度与调度（js/engine/scheduler.js）

学员画像新增 `mastery: { [skillId]: { window: 最近20题正误环形数组, speedOk, state: learning|review|maintain, lastSet, interval, nextDue } }`。

日课选题算法：
1. 口算页：地基层按（未达标 > 维持到期 > 维持未到期）加权抽 40 题；
2. 主题页：选 2-3 个模块 = 在练技能点（按 prereqs 就绪 + gradeBand 排序取前 2）+ 1 个回访模块（interval 到期的已掌握点）；模块内题目交错排列；
3. 回炉页：沿用 v2 压力分三档（0/5/8 题），变式走 bug 阶梯。
所有抽样用 v2 的种子化 rng + 盖章机制（不变），保证补打与跨设备一致。

## 5. 双形态组卷（js/engine/composer.js）

- **日课单**（默认）：口算计时页（含秒表格与上次成绩）+ 交错主题页 + 回炉页（按需），每人 2-3 页 A4；
- **拟真卷**（手动按钮）：按学员地图位置组 100 分六大题卷（结构继承 v2 版面），限时 40 分钟，用于周测/月考验收；
- 两种形态同一题模库、同一批阅回写、同一打印引擎。

## 6. 错题本 v3（js/engine/errorbook.js 升级）

条目在 v2 基础上增加 `bugId?`、`skill`、`variantHistory[]`（出过的变式指纹，防短期重复）。

出卷双通道：
- **一键预设**：到期复练 / 复错优先 / 本周错题周清 / 按技能点专项 —— 点击 → 预览 → 打印，两次点击；
- **自由筛选**：学员 × 技能域 × bug × 时间窗 × 掌握状态组合，实时预览；
- **训练册规格**：每条错题 = 原题 + L2 换数变式 + L3 结构变式（复错题追加 L4 陷阱对比题），A4 自适应排版 + 独立答案页 + 批改勾选框。

## 7. 数据迁移与同步

- v2 错题本自动迁移：旧 tag → 新 skillId 映射表（js/migrate.js），count/日期/掌握状态保留；旧 difficulty 映射为各技能点初始 state；
- Gist 数据加 `schemaVersion: 3`，拉到旧版自动升级后回写；合并策略沿用（并集/取最大/取最新/盖章先者胜），mastery 合并按 lastSet 较新一方；
- localStorage 前缀不变（MMC2），新增 key 由 store.js 统一管理。
